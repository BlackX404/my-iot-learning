# Tag 里程碑：为什么打 tag，怎么打

Git tag 就是把某个 commit 钉死，给它一个名字。跟分支不同——分支会随着你继续 commit 往前走，tag 永远不动。

**在这个计划里：** 每周六完成验证后打一个 tag，就像在日历上画一个勾。

```
week-1  ← 第 1 周完成
week-2  ← 第 2 周完成
week-3  ← 第 3 周完成
...
v1.0    ← 第 7 周：CRAIC 记录器第一版（正式版本标记）
```

四年后回到这个仓库，点上某个 tag（比如 `week-1`），就能看到 2026 年 7 月第一周的代码。不是回忆，是实打实的版本快照。

**两种 tag：**

| 类型 | 命令 | 用途 |
|---|---|---|
| 轻量 tag | `git tag week-1` | 个人标记，够用 |
| 带注释 tag | `git tag -a v1.0 -m "说明"` | 版本发布，给别人看的 |

每周用轻量版 `git tag week-N`。第 7 周项目完成后用带注释的 `git tag -a v1.0 -m "..."`。

# Git 速查表

```bash

# 每天用
git status              # 看当前状态
git add -A              # 暂存所有修改
git commit -m "信息"    # 提交
git push                # 推到 GitHub
git pull                # 拉取远程更新
git log --oneline       # 看提交历史

# 分支
git branch 分支名       # 创建分支
git checkout 分支名     # 切换分支
git merge 分支名        # 合并分支到当前

# 回退
git stash               # 暂存未提交的修改
git stash pop           # 恢复暂存的修改
git reset --soft HEAD~1 # 撤销最近一次 commit（保留修改）

# 看差异
git diff                # 看未暂存的修改
git diff --staged       # 看已暂存的修改

# 打标签
git tag week-1          # 轻量标签（个人标记）
git tag -a v1.0 -m "说明"  # 带注释标签（正式版本）
git tag                 # 列出所有标签
git push --tags         # 推送所有标签到 GitHub
```
